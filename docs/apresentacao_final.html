PARTE 1 DE 3

O Cenário

O que o trabalho pediu — e por que era um problema real, não só um exercício.

O problema

Dois sistemas que nunca conversaram entre si

O Estado do Ceará publica dados de execução orçamentária — empenhos, ordens bancárias, unidades gestoras — em um banco PostgreSQL. Os contratos públicos assinados vivem em uma API REST separada, no portal Ceará Transparente. Cada fonte, isolada, já é grande: mais de 1,4 milhão de empenhos e 617 mil contratos desde 2007.

Sem cruzar as duas, nenhum órgão de controle consegue responder, de forma sistemática, se o dinheiro contratado está sendo pago corretamente, se há sinais de irregularidade, ou quanto será preciso reservar de orçamento no próximo trimestre.

O enunciado pedia respostas para 3 perguntas

Rastreabilidade, anomalias e previsão

  1. Rastreabilidade financeira — de um contrato assinado, qual % do valor já foi efetivamente pago via ordem bancária, e qual está apenas empenhado? Existem contratos pagos acima do valor contratado?
  2. Detecção de anomalias — há contratos de alto valor por dispensa ou inexigibilidade de licitação? Credores com histórico de infração estão recebendo novos contratos?
  3. Previsão e planejamento — com base no histórico, dá para estimar o volume de pagamentos do próximo trimestre, por órgão?
Pipeline Ceará Transparente04 / 18
O que o enunciado exigia

Um sistema de dados completo, de ponta a ponta

Não bastava responder as perguntas manualmente uma vez — o trabalho pedia um pipeline reproduzível, capaz de:

Extrair dos dois sistemas de forma incremental, por data
Unificar tudo em um modelo dimensional (Data Warehouse)
Guardar os dados brutos e transformados no HDFS (Hadoop)
Orquestrar todo o fluxo via Apache Airflow, com DAGs agendadas
Treinar modelos de ML para anomalias e previsão de pagamentos
Gerar um relatório automático com os insights dos modelos

Passo 2 · Extração incremental

Nunca reprocessar o que já foi extraído

①

Postgres de origem

Lê empenhos, ordem_bancaria_orcamentaria e unidade_gestora filtradas por data, salva como Parquet na Bronze.

②

API de contratos

Pagina os contratos assinados no período, respeitando o total de páginas retornado pela própria API, salva cada página como JSON na Bronze.

③

Marcador incremental

Uma Airflow Variable guarda até onde já foi extraído — a próxima execução começa exatamente dali, sem reprocessar nem perder dados.

Passo 4 · Orquestração

4 DAGs que se disparam sozinhas, em cadeia

Sem horário fixo entre elas nem sensor externo — cada DAG publica um "Dataset" ao terminar, e isso dispara a próxima automaticamente (recurso nativo do Airflow 2.x).

  • dag_bronze_extract → publica o Dataset Bronze success
  • dag_silver_transform → disparada pelo Dataset Bronze, publica o Dataset Silver success
  • dag_gold_load → disparada pelo Dataset Silver, publica o Dataset Gold success
  • dag_ml_inference → disparada pelo Dataset Gold, treina os modelos e gera o relatório success

Passo 5 · Machine Learning

Dois modelos, dois tipos de pergunta

①

Isolation Forest — detecção de anomalias

Não supervisionado: aprende o "normal" a partir de valor, modalidade, tipo de objeto, vigência, emergência e histórico do credor, e pontua cada contrato de 0 a 1.

②

XGBoost quantílico — previsão de pagamentos

Prevê o volume de pagamentos do próximo trimestre por órgão, com intervalo de confiança (p10/p50/p90) — e cai para uma previsão mais simples quando o histórico é curto demais.

Passo 7 · Qualidade e testes

14 testes automatizados — e 3 bugs que só apareceram com dados reais

Testes com dados sintéticos garantem a lógica; rodar contra o volume real revelou problemas que só existem em escala:

  • Timezone nas datas da API — datas de contrato vinham com fuso explícito e quebravam ao juntar com datas do Postgres, que não têm fuso.
  • Órgãos que mudam de "dono" — 7 unidades gestoras trocaram de órgão superior entre anos, duplicando linhas de empenho no cruzamento.
  • Extração incremental mal consolidada — a camada intermediária só lia a última execução da extração, perdendo o histórico acumulado.

Os três viraram testes de regressão — não vão voltar a acontecer sem avisar.
